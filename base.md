# Travel Checklist App – Content Guide

이 문서는 **Travel Checklist App**를 기획할 때 고려했던 **기능 전체 목록**, **콘텐츠 구조**, 그리고 **JSON 데이터 설계 예시**를 한 번에 정리한 가이드입니다.

---

## 1. 앱의 핵심 목적

- 여행 전/중/후 필요한 준비물을 **체크리스트 형태**로 관리
- 여행 유형, 국가, 기간, 동반자에 따라 **맞춤 체크리스트 자동 생성**
- 사용자가 직접 항목을 추가/수정/공유 가능

---

## 2. 주요 기능 전체 목록

### 2.1 여행 생성 & 관리

- 여행 생성

  - 여행 이름
  - 출발일 / 귀국일
  - 목적지 (국가, 도시)
  - 여행 유형 (해외 / 국내 / 출장 / 배낭 / 가족)
  - 동반자 유형 (혼자 / 커플 / 가족 / 아이 포함)

- 여행 수정 / 삭제

- 다중 여행 관리 (여러 여행 동시 보관)

---

### 2.2 기본 체크리스트 자동 생성

- 여행 생성 시 조건 기반 자동 생성

  - 해외 여행 → 여권, 비자, 환전
  - 겨울 여행 → 외투, 장갑
  - 출장 → 노트북, 충전기

- 카테고리별 구성

  - 필수 서류
  - 의류
  - 전자기기
  - 세면/위생
  - 건강/의약품
  - 기타

---

### 2.3 체크리스트 관리

- 체크 항목 상태

  - 미완료
  - 완료

- 항목 기능

  - 체크/언체크
  - 개별 삭제
  - 순서 변경 (Drag & Drop)
  - 중요 표시 (⭐)

- 사용자 커스텀 항목 추가

---

### 2.4 카테고리 관리

- 기본 카테고리 제공
- 사용자 정의 카테고리 생성
- 카테고리별 접기/펼치기

---

### 2.5 반복 & 템플릿 기능

- 자주 쓰는 여행 템플릿 저장
- 이전 여행 체크리스트 복사
- 출장이 잦은 사용자용 프리셋

---

### 2.6 알림 & 리마인더

- 출발 전 알림

  - 7일 전
  - 3일 전
  - 하루 전

- 특정 항목 미완료 알림

- 사용자 지정 알림 시간

---

### 2.7 공유 기능

- 체크리스트 공유 링크 생성
- 동반자와 공동 체크
- 읽기 전용 / 수정 가능 권한

---

### 2.8 오프라인 지원

- 네트워크 없이 체크 가능
- 온라인 복구 시 자동 동기화

---

### 2.9 다국어 & 지역화

- 언어 지원

  - 한국어
  - 영어
  - 일본어 (확장 가능)

- 국가별 추천 항목 다름

---

## 3. 데이터 구조 (JSON 예시)

### 3.1 여행 데이터

```json
{
  "tripId": "trip_001",
  "title": "Tokyo Business Trip",
  "destination": {
    "country": "Japan",
    "city": "Tokyo"
  },
  "startDate": "2026-03-01",
  "endDate": "2026-03-05",
  "type": "BUSINESS",
  "companions": "ALONE"
}
```

---

### 3.2 체크리스트 카테고리

```json
{
  "categoryId": "cat_documents",
  "name": "Documents",
  "order": 1
}
```

---

### 3.3 체크리스트 아이템

```json
{
  "itemId": "item_001",
  "categoryId": "cat_documents",
  "title": "Passport",
  "checked": false,
  "important": true,
  "createdBy": "system"
}
```

---

### 3.4 사용자 커스텀 아이템

```json
{
  "itemId": "item_custom_001",
  "categoryId": "cat_etc",
  "title": "Extra charger",
  "checked": false,
  "createdBy": "user"
}
```

## 6. 정리

이 구조를 기반으로 하면:

- Flutter / React Native / Web 모두 대응 가능
- JSON 기반 API 설계가 단순해짐
- 템플릿 + 커스텀 확장이 쉬운 구조

원하면 다음 단계로 **DB 스키마**, **API 명세**, **Flutter MVVM 구조**까지 바로 이어서 정리할 수 있음 👍
